# OSS Export 公開準備チェックリスト

## 公開前に実行すること

### 1. シンボリックリンクの削除
```bash
cd oss-export
rm .planning  # シンボリックリンクを削除
```

### 2. 不要なファイルの削除
```bash
# .git/ があれば削除（既存リポジトリの履歴をクリア）
rm -rf .git

# 親リポジトリ固有のファイルを削除
rm -f ../.env.local
rm -f ../.claude/
rm -f oss-export/.claude/settings.local.json
```

### 3. package.json の最終確認
- ✅ `name`: "yarikiru-oss"
- ✅ `version`: "5.0.0"
- ✅ `private`: false
- ✅ `bin`: {"yarikiru": "./cli/index.mjs"}

### 4. README.md の最終確認
- ✅ インストール方法
- ✅ クイックスタート
- ✅ CLIコマンド一覧
- ✅ 技術スタック
- ✅ ライセンス

### 5. .gitignore の確認
```
.env.local
.env.*.local
.next/
turso/
.planning/  # ← 重要！.planning は含めない
.claude/
```

### 6. 新しいGitリポジトリの初期化
```bash
cd oss-export
rm -rf .git  # 既存のgit履歴を削除
git init
git add .
git commit -m "Initial release of yarikiru-oss v5.0.0"

# GitHubに新しいリポジトリを作成してプッシュ
git remote add origin https://github.com/045team/yarikiru-oss.git
git branch -M main
git push -u origin main
```

### 7. npm パブリッシュ（オプション）
```bash
npm publish
```
