# Git Commit/Push Rules — Agentic Workflow

## 基本原則
- **1タスク = 1コミット**: GSDの各サブタスクが完了（`[x]` にチェック）されたらコミットする。
- **大きな変更は分割**: 1ファイルに100行以上の変更がある場合、論理的に分割可能か検討する。

## コミットメッセージ規約

```
[TaskID] emoji 日本語の要約

例:
[g_1771859322453] ✅ OSS用ダッシュボードUIの新規作成
[s_1771859322535] 🔧 CLIエントリーポイントをローカルSQLite専用に書き換え
[HOTFIX] 🚑 ビルドエラーの修正
```

### Emoji Prefix
| Emoji | 用途 |
|-------|------|
| ✅ | 新機能完了 |
| 🔧 | リファクタ / 改善 |
| 🚑 | ホットフィックス |
| 📝 | ドキュメント更新 |
| 🧹 | 不要コード削除 |
| 🧪 | テスト追加・修正 |

## Push 規約
- **main ブランチへの直接pushは禁止** (OSS公開後)
- feature ブランチ → PR → merge の流れを基本とする
- ただし、開発初期フェーズ（v5.0構築中）では main への直接pushを許可

## エージェント向けルール
- エージェントは作業完了時に **必ず** `git add -A && git commit` を行うこと
- コミットメッセージにはタスクIDを含めること
- `git push` はエージェント単独では行わず、ユーザーの承認を得ること
